# JoyBoy

Repositorio de práctica para aprender el flujo de Pull Requests en GitHub.

## ¿Qué es esto?

Este repo existe para practicar el ciclo completo: crear una rama, hacer un cambio, y abrir un Pull Request.

## Flujo de un Pull Request

1. Crear una rama nueva a partir de `main`.
2. Hacer el cambio y confirmarlo (`commit`).
3. Subir la rama (`push`).
4. Abrir el Pull Request y esperar revisión.
5. Fusionar (`merge`) cuando esté aprobado.
